# 一斉メール送信システム

GoogleスプレッドシートとGmail APIを活用した、パーソナライズHTMLメールの一斉送信Webアプリです。

## 機能

- 📊 **スプレッドシート読み込み**: URLを貼るだけで宛先リストを自動取得
- ✍️ **メール作成**: リアルタイムHTMLプレビュー付き、画像添付対応
- 📨 **テスト送信**: 本番前に実際にメールを送信して確認
- 🚀 **下書き一斉作成**: 全宛先分の下書きをGmail APIで自動作成、進捗リアルタイム表示
- 📅 **送信予約**: Gmailの「送信日時を指定」機能で一括予約送信

## セットアップ手順

### 1. Google Cloud Console の設定

#### (1) プロジェクト作成
1. [Google Cloud Console](https://console.cloud.google.com/) にアクセス
2. 上部の「プロジェクトを選択」→「新しいプロジェクト」をクリック
3. プロジェクト名を入力して作成

#### (2) APIの有効化
「APIとサービス」→「ライブラリ」で以下の2つを検索して有効化:
- **Google Sheets API**
- **Gmail API**

#### (3) OAuth 2.0 クライアントIDの作成
1. 「APIとサービス」→「認証情報」→「認証情報を作成」→「OAuth クライアント ID」
2. アプリの種類: **ウェブアプリケーション**
3. 承認済みリダイレクト URI に以下を追加:
   ```
   http://localhost:3000/auth/callback
   ```
4. 作成後、**クライアントID** と **クライアントシークレット** をメモ

#### (4) OAuth 同意画面の設定（未設定の場合）
1. 「APIとサービス」→「OAuth 同意画面」
2. ユーザーの種類: **外部**（または内部）
3. アプリ名、サポートメール、デベロッパー連絡先を入力
4. スコープの追加:
   - `https://www.googleapis.com/auth/gmail.compose`
   - `https://www.googleapis.com/auth/spreadsheets.readonly`
   - `https://www.googleapis.com/auth/userinfo.email`
5. テストユーザーにご自身のGoogleアカウントを追加

---

### 2. プロジェクトのセットアップ

#### (1) `.env` ファイルの作成
`.env.example` をコピーして `.env` を作成し、実際の値を設定します:

```bash
cp .env.example .env
```

`.env` を編集:
```env
GOOGLE_CLIENT_ID=取得したクライアントID
GOOGLE_CLIENT_SECRET=取得したクライアントシークレット
GOOGLE_REDIRECT_URI=http://localhost:3000/auth/callback
SESSION_SECRET=任意のランダムな文字列（例: my-super-secret-key-2024）
PORT=3000
```

#### (2) 依存パッケージのインストール
```bash
npm install
```

#### (3) サーバーの起動
```bash
npm start
```

または開発時（ファイル変更を自動検知）:
```bash
npm run dev
```

#### (4) ブラウザでアクセス
```
http://localhost:3000
```

## 使い方

### Step ①: 宛先読み込み
1. 「Googleでログイン」ボタンをクリックして認証
2. スプレッドシートのURLを入力して「読み込む」

**スプレッドシートの形式:**
| 名前 | メールアドレス |
|------|-------------|
| 山田太郎 | yamada@example.com |
| 鈴木花子 | suzuki@example.com |

- 1行目はヘッダー行（「名前」「メールアドレス」など）
- 「名前」列: name、氏名、姓名 なども自動検出
- 「メールアドレス」列: email、mail、address なども自動検出

### Step ②: メール作成
1. 件名を入力
2. 本文を入力（送信時に `{名前}様` に自動置換されます）
3. 必要に応じて画像を添付（ドラッグ&ドロップ対応）
4. 右側のプレビューでレイアウトを確認

### Step ③: テスト送信
1. テスト送信先メールアドレスを入力（省略時はログインアカウント宛）
2. 「テストメールを送信」をクリック
3. メールを受信して内容を確認
4. 「確認しました」チェックボックスにチェック

### Step ④: 下書き一斉作成
1. 「下書きを一斉作成する」ボタンをクリック
2. 進捗バーでリアルタイム確認（100ms間隔で順次作成）
3. 完了後、「Gmailの下書きを開く」リンクからGmailへ
4. 下書きを選択 → 「送信日時を指定」で予約送信

## スプレッドシートの共有設定

スプレッドシートは以下のいずれかの方法でアクセス可能にしてください:
- ログインしているGoogleアカウントがオーナーまたは閲覧者として追加されている
- または「リンクを知っている全員が閲覧可」に設定

```
auto-mail/
├── CLAUDE.md          # プロジェクト仕様書
├── .env               # 環境変数（要作成、gitignore対象）
├── .env.example       # 環境変数テンプレート
├── .gitignore         # .envとnode_modules/を除外
├── package.json
├── server.js          # Expressサーバー + APIエンドポイント
├── auth.js            # Google OAuth2 認証モジュール
├── sheets.js          # Google Sheets API モジュール
├── gmail.js           # Gmail API モジュール
└── public/
    └── index.html     # フロントエンド（単一ファイル）
```

## 注意事項

- **`.env` は絶対にGitリポジトリにコミットしないでください**（`.gitignore` で除外済み）
- Gmail APIは1秒あたり約10リクエストの制限があります（100ms間隔で対処済み）
- `gmail.compose` スコープのみ使用（受信トレイへのアクセスなし）
- 本番環境ではHTTPS必須（localhost開発時はHTTP可）
- OAuth同意画面が「テスト」状態の場合、テストユーザーとして登録したGoogleアカウントのみログイン可能

## トラブルシューティング

| エラー | 対処法 |
|--------|--------|
| `GOOGLE_CLIENT_ID が設定されていません` | `.env` ファイルを作成して環境変数を設定 |
| `スプレッドシートへのアクセス権がありません` | スプレッドシートの共有設定を確認 |
| `スプレッドシートが見つかりません` | URLまたはIDを確認。コピーミスがないか確認 |
| `「名前」列が見つかりません` | ヘッダー行に「名前」「name」などの列名を追加 |
| OAuth エラー | Google Cloud ConsoleでリダイレクトURIが正しく設定されているか確認 |
| `アクセスがブロックされました` | OAuth同意画面でテストユーザーに追加されているか確認 |
